# 26_06_29 账本项目 - 理解 Node.js、npm 与 node_modules

## 项目背景

这次学习发生在 `Local-First Trading Ledger` 账本项目。场景是阅读项目根目录时，看到很多不熟悉的文件和目录：

- `node_modules/`
- `.next/`
- `.git/`
- `package.json`
- `package-lock.json`
- `tsconfig.json`
- `next.config.mjs`
- `tailwind.config.ts`
- `vitest.config.ts`
- `.eslintrc.json`
- `src/`

真正卡住的不是某一段业务代码，而是看不懂 JavaScript/TypeScript 工程根目录里哪些该优先读、哪些只是工具配置、哪些是生成物或第三方依赖。

## 当时卡住的问题

我把很多文件都当成“需要理解的代码”。尤其看到 `node_modules/` 里面有大量目录时，会本能地怀疑：这些是不是也都要读懂？

这会带来一个严重问题：以后从 GitHub 上 `git clone` 别人的项目时，如果不能先识别 [[JavaScript Project Structure（JavaScript 项目结构）]]，就会一头扎进不该读的文件里，越看越乱。

## 学到的内容

这次把几个经常混在一起的概念拆开了：

- [[JavaScript Runtime（JavaScript 运行时）]]：代码运行时所在的环境。
- [[Node.js（Node 运行环境）]]：让 JavaScript 离开浏览器，在本地、服务器和命令行里运行。
- [[V8（JavaScript 引擎）]]：Node.js 用来执行 JavaScript 的 engine。
- [[npm（Node 包管理器）]]：安装 package、管理依赖、执行项目 scripts。
- [[Package（软件包）]]：别人写好的、可以安装进项目复用的代码工具包。
- [[package.json（项目清单）]]：项目说明、命令菜单和依赖清单。
- [[package-lock.json（依赖锁文件）]]：依赖精确版本和完整依赖树。
- [[Dependency（依赖）]]：项目运行、开发、测试或构建需要的 package。
- [[node_modules（依赖实体目录）]]：npm 下载到本地的第三方依赖实体目录。

这次之后，继续学习了 [[26_06_29 账本项目 - 建立 React 和 Next.js 的基础心智模型]]，把 `next`、`react`、`typescript`、`tailwindcss` 这些 package 进一步放回前端应用分层里理解。

## 现在的判断规则

拿到 JavaScript/TypeScript 项目时，不要一上来乱读。先分层：

第一层：优先看。

- `README.md`
- [[package.json（项目清单）]]
- `src/`

第二层：知道用途，遇到对应问题再深入。

- `tsconfig.json`
- `next.config.mjs`
- `tailwind.config.ts`
- `postcss.config.mjs`
- `.eslintrc.json`
- `vitest.config.ts`
- `next-env.d.ts`
- [[package-lock.json（依赖锁文件）]]
- `.gitignore`

第三层：基本别作为读项目入口。

- [[node_modules（依赖实体目录）]]
- `.next/`
- `.git/`
