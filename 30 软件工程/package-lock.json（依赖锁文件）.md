# package-lock.json（依赖锁文件）

## 一句话

`package-lock.json（依赖锁文件）` 记录 npm 实际安装的精确依赖版本和依赖树，用来提高不同机器安装结果的一致性。

## 背景：它出现前的问题

[[package.json（项目清单）]] 通常只表达项目需要哪些 package 和允许的版本范围。不同时间、不同机器安装依赖时，如果只看版本范围，可能得到不同的依赖树，造成“我这里能跑，你那里不能跑”。

## 它解决了什么

package-lock.json 把“项目需要哪些工具”进一步固定成“这次到底精确安装了哪些版本的工具”，包括依赖的依赖。

它帮助团队尽量保证：

- 本机安装的依赖版本一致。
- 其他开发者安装的依赖版本一致。
- CI/CD 环境安装的依赖版本一致。

## 核心机制

依赖锁文件会记录：

- 直接依赖的精确版本。
- 传递依赖的精确版本。
- 下载来源。
- 完整依赖树。

通常开发者不需要手动阅读或编辑它，但它应该和 package.json 一起提交到 Git。

## 典型使用场景

- 让 `npm install` 尽量复现同一套依赖。
- 排查依赖版本差异导致的问题。
- 在团队协作和部署环境中固定依赖树。

## 局限性

package-lock.json 不是业务源码，也不是初学读项目的主入口。读项目时应先看 README、[[package.json（项目清单）]] 和 `src/`，遇到依赖版本问题时再深入锁文件。

## 和其他概念的关系

- [[npm（Node 包管理器）]] 生成并使用 package-lock.json。
- [[Dependency（依赖）]] 的精确安装结果会进入锁文件。
- [[node_modules（依赖实体目录）]] 是锁文件描述的依赖树在本地的实体结果。
- [[JavaScript Project Structure（JavaScript 项目结构）]] 把 package-lock.json 归为“知道用途即可，遇到问题再深入”的配置类文件。
