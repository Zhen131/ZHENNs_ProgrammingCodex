# 为什么 .next 不用读

## 痛点

Next.js 项目根目录里可能有 `.next/`。它看起来像一个重要目录，但初学者如果直接进去读，很容易把构建产物和业务源码混在一起。

## 以前怎么解决

没有项目结构分层意识时，看到目录就想打开读。但 `.next/` 不是人手写的源码，也不是 Next.js 工具包本体。

## 技术演进

需要区分几个位置：

```text
src/               我写的源码
node_modules/next/ Next.js 这个工具包实体
next.config.mjs    Next.js 配置
.next/             Next.js 加工项目后生成的产物和缓存
```

因此 `.next/` 的判断规则是：

```text
.next/ 不读
.next/ 不手改
.next/ 不提交
.next/ 可以删除后重新生成
```

## 我的理解

`.next/` 是 Next.js 运行或构建后的结果，不是学习项目的入口。读项目仍然应该优先看 README、[[package.json（项目清单）]] 和 `src/`。
