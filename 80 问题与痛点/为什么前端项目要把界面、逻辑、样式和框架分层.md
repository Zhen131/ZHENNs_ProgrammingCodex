# 为什么前端项目要把界面、逻辑、样式和框架分层

## 痛点

读现代前端项目时，会同时遇到 React、TypeScript、TSX、CSS、Tailwind、Next.js、路由、布局、构建和配置。初学者很自然会问：为什么不能全部写在一起？

## 以前怎么解决

简单页面可以把结构、样式和脚本混在一起。但项目变复杂后，全部写在一起会让文件越来越大，职责越来越混乱，改一个地方容易影响很多东西。

## 技术演进

这次可以先建立一个临时分层：

- [[React（React UI 库）]] / [[UI Component（UI 组件）]]：负责界面零件。
- [[TypeScript（类型化 JavaScript）]]：负责逻辑和类型。
- [[JSX（JavaScript XML）]] / [[TSX（TypeScript XML）]]：负责在代码中描述界面结构。
- [[CSS（层叠样式表）]] / [[Tailwind CSS（Tailwind 样式工具）]]：负责样式。
- [[Next.js（Next 应用框架）]]：负责应用结构、路由、布局、构建和运行规则。

## 乐高类比

可以把这套前端技术先理解成一个 Lego（乐高）作品的制作流程：

- [[React（React UI 库）]] 像乐高零件系统，核心思想是把界面拆成可组合的 [[UI Component（UI 组件）]]。
- Component 像一块块乐高 piece：可以是按钮、表单、列表、卡片，也可以继续由更小的 component 组合出来。
- JavaScript / [[TypeScript（类型化 JavaScript）]] 更像零件的行为和数据规则：这个零件接收什么数据、点击后做什么、状态如何变化。
- [[JSX（JavaScript XML）]] / [[TSX（TypeScript XML）]] 更像零件的结构说明：这个零件里面有哪些按钮、文字、输入框和子零件。
- [[CSS（层叠样式表）]] / [[Tailwind CSS（Tailwind 样式工具）]] 像给零件上颜色、贴装饰、定尺寸和摆位置。
- [[Next.js（Next 应用框架）]] 像说明书、底板和施工规则：它规定这些 React 零件放在哪个页面、怎么路由、怎么布局、怎么运行和构建成完整应用。

这个类比的关键不是把每个词背死，而是先分清“零件本身”“零件行为”“零件长相”“零件样式”和“作品组织规则”不是同一层东西。

## 我的理解

分层的目的不是让初学者多记名字，而是让复杂应用的不同问题有不同位置：界面怎么拆、行为怎么写、结构怎么描述、样式怎么控制、页面怎么路由、项目怎么构建。现在我还没有完全理解每一层的历史原因，但先用乐高类比记住它们不是同一类东西。
